# Topic Notes

Use this directory for one Markdown file per chat or work topic.

## Naming rule

- File name format: `YYYY-MM-DD-<slug>.md`
- Use a short slug that describes the topic, not the task status.
- Keep each file focused on one workstream or conversation thread.

## Required structure

- Start from [`TEMPLATE.md`](TEMPLATE.md).
- Link related specs from `SPECS/triage/`.
- List the `AGENTS.md` constraints that shaped the note.
- If a note conflicts with `AGENTS.md`, the note is invalid until reconciled.
